# SDK（X SDK）复杂场景参考

## 适用范围

- 与 LLM 后端集成（流式输出）
- 统一消息/工具协议
- 前后端职责划分

## 关键原则

- 前端仅负责渲染与交互，业务协议与权限由服务端控制。
- 流式输出应追加内容，避免全量替换造成滚动跳动。
- 工具调用结构化存储，便于审计与回放。

## 常见实践

- 流式状态：维护 `streamingMessageId` 与 `isGenerating`。
- 错误处理：错误也作为消息实体展示。
- 取消与重试：必须具备可中断机制。
