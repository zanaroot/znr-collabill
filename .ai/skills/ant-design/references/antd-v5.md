# Ant Design v5 参考

## 适用场景边界
- 存量项目仍停留在 `antd@^5`，无法立即升级。
- 需要评估 v6 迁移成本与风险的维护期项目。

## 推荐模式（1–2 种）
1. 维护优先：功能修复与小改动为主，避免引入新的主题体系或样式依赖。
2. 迁移准备：逐步清理全局样式覆盖与内部 class 依赖，为 v6 迁移做准备。

## 必须避免的反模式
- 新功能完全按 v5 写法扩展而不评估迁移成本。
- 引入新的 `.ant-*` 全局覆盖或内部 class 依赖。
- 忽略 v6 迁移清单中的破坏性变更。

## 最小示例
```ts
// 迁移准备清单（示意）
const checklist = [
  "移除全局 .ant-* 覆盖",
  "替换内部 class 依赖",
  "对照 migration-v6 清单逐项核对",
];
```

## 与主 Skill 的回跳说明
- 当只需判断“是否维持 v5 维护或启动迁移”时，回到主 Skill 的决策与分流规则。
